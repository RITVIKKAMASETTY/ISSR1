## Project Overview

This project implements a complete audio processing pipeline for team communication analysis in human-factors simulated environments. The pipeline processes raw meeting recordings through a structured 4-stage enhancement system, preparing them for downstream tasks including automatic speech recognition (ASR), speaker diarization, and communication pattern analysis.

### Problem Statement

Raw meeting audio contains multiple challenges that degrade downstream analysis:

| Challenge | Impact |
|-----------|--------|
| **Background Noise** | HVAC, equipment hum, ambient sounds mask speech |
| **Channel Mismatch** | Multi-channel recordings require standardization |
| **Inconsistent Loudness** | Variable recording levels affect ASR accuracy |
| **Sample Rate Variation** | Different rates break model compatibility |

### Solution Overview

Two-notebook system providing:
1. **Exploratory Analysis** — Understanding audio characteristics
2. **Enhancement Pipeline** — 4-stage processing for clean, normalized output

### Stage Details

| Stage | Operation | Parameters | Problem Solved | Critical Ordering |
|-------|-----------|------------|----------------|------------------|
| **1** | Mono Conversion | `librosa.to_mono()` | Multi-channel inconsistency | First — cheaper resampling |
| **2** | Resampling | 16,000 Hz target | ASR compatibility | Before noise reduction (STFT sensitive) |
| **3** | Noise Reduction | `prop_decrease=0.8`, `stationary=False` | Background noise | After resampling (correct sample rate) |
| **4** | Peak Normalization | `peak_level=0.98` | Inconsistent loudness | Final stage (clean signal) |

### Spectral Gating Explained
Input Audio
│
▼
┌─────────────────┐
│ STFT (FFT) │ ──▶ Time-frequency representation
└─────────────────┘
│
▼
┌─────────────────┐
│ Noise Floor │ ──▶ Estimate from quietest frames (20% percentile)
│ Estimation │
└─────────────────┘
│
▼
┌─────────────────┐
│ Masking │ ──▶ Suppress frequencies below noise floor × threshold
│ (prop=0.8) │ (80% suppression avoids musical noise)
└─────────────────┘
│
▼
┌─────────────────┐
│ Inverse STFT │ ──▶ Reconstruct denoised time-domain signal
└─────────────────┘

text

### Why `prop_decrease = 0.8`?

| Value | Effect |
|-------|--------|
| 0.0 | No noise reduction |
| 0.8 | **Selected** — 80% suppression; balances noise removal with speech preservation |
| 1.0 | Full suppression; creates musical noise artifacts (discontinuous tones) |

Metric Definitions
Metric	Formula	Interpretation
SNR	$10 \cdot \log_{10}\left(\frac{P_{\text{speech}}}{P_{\text{noise}}}\right)$	Higher = less noise
RMS	$\sqrt{\frac{1}{N}\sum x_i^2}$	Average loudness
Dynamic Range	$20 \cdot \log_{10}\left(\frac{\text{peak}}{\text{RMS}}\right)$	Peak-to-RMS ratio; higher = more expressive
Crest Factor	$20 \cdot \log_{10}\left(\frac{\text{peak}}{\text{RMS}}\right)$	Same as dynamic range
ZCR	$\frac{1}{T-1}\sum_{t=1}^{T-1} \mathbf{1}{{x_t \cdot x{t-1} < 0}}$	High-frequency proxy
High-Freq Energy	$\frac{\sum_{f > 4\text{kHz}}	X_f	^2}{\sum_f	X_f	^2} \times 100%$	Noise indicator
